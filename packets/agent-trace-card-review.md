# Review Packet: agent-trace-card

## Problem Statement

Agent traces are hard to share because raw logs can be noisy, sensitive, and
missing decision context.

## Repo

`github.com/auraoneai/agent-trace-card`

## Example Command

```bash
agent-trace-card generate examples/support_trace.json --out card.md
```

## Narrow Technical Ask

Does this card capture the minimum evidence you would need to debug or review an
agent failure without exposing raw production logs?
